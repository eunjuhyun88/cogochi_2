# Decision Records

Purpose:
- store stable, reviewable decisions that other agents can trust without rereading full chat history

Use this folder for:
- IA and route naming decisions
- architectural boundaries
- state ownership changes
- collaboration workflow changes

Rules:
- one file per decision
- keep the filename sortable
- include context, decision, impact, and rollback notes
- update a prior decision instead of creating contradictory silent drift

Suggested naming:
- `0001-short-slug-YYYY-MM-DD.md`
