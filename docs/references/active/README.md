# Active References

- Keep still-used support docs out of the root `docs/` surface.
- Preserve deep technical context without treating these files as canonical authority.

Use this folder for:
- dated execution designs still referenced by canonical docs
- security, passport, terminal, and arena support specs
- audits and boundary maps still used by active plans

Do not use this folder for:
- new canonical product or architecture decisions
- one-off brainstorming drafts
- generated reports
