# Core Beliefs

Purpose:
- Stable agent-first beliefs for how this repo should manage context and design.

## Beliefs

1. Give the agent a map, not an encyclopedia.
2. Canonical docs should be short, stable, and composable.
3. Product intent must be visible from repo-local markdown.
4. One canonical implementation target is better than many drifting clones.
5. Stable rules belong in docs or scripts, not only in chat or watch logs.
6. Historical material is reference, not authority.
7. State ownership should be explicit and minimal.
8. Server-derived truth should not be reauthored by the client.
9. Small entrypoint docs should point to deeper sources of truth.
10. Repeated human judgment should be promoted into reusable scaffolding.

## Anti-Patterns

- one giant instruction file
- dated working docs acting as silent canon
- archive docs mixed with active docs
- sibling clone duplication as normal workflow
- watch log as the only memory system
