## 1. 이 문서의 역할

이 문서는 **engagement(재미/중독)가 아니라 usability(편의/효율)** 관점에서 전체 제품을 설계한다.

구분:
- **Usability** = 유저가 원하는 걸 얼마나 빨리, 적은 마찰로 할 수 있는가 ← **이 문서**
- **Engagement** = 유저가 얼마나 자주, 오래 머무는가 ← `UserJourney v1` 참고 (별도 레이어)
- **Architecture** = 코드가 어떻게 나뉘는가 ← `full-system-redesign-blueprint` 참고

## 2. UX 원칙 (6개, 모든 페이지에 적용)

### P1. Action First — 행동이 읽기보다 먼저

모든 화면의 첫 번째 시각 요소는 "읽을 것"이 아니라 "할 수 있는 것"이다.
- 나쁜 예: 차트 → 지표 목록 → 스크롤 → 하단에 트레이드 버튼
- 좋은 예: 차트 + 즉시 접근 가능한 액션 바 (Track / Trade / Share)

적용 검증: 각 페이지에서 "유저가 주요 액션 버튼까지 몇 초/몇 클릭이 필요한가?" 측정.

### P2. Explain Always — 모든 추천/수치에 이유 표시

숫자나 추천을 보여줄 때 반드시 1줄 이유를 함께 보여준다.
- 나쁜 예: "Entry Score: 74"
- 좋은 예: "Entry Score: 74 — OI 과열 + 고래 유입 둔화"

적용 검증: 화면에 수치가 있으면 바로 아래/옆에 이유 텍스트가 있는가?

### P3. Risk Visible — 손실 가능성은 항상 같은 위치

리스크 정보를 숨기지 않는다. 항상 고정 위치에 표시.
- 포지션이 열려 있으면: 현재 PnL + 손절 거리가 항상 보임
- 에이전트 추천이면: 최근 실패율도 같이 보임

적용 검증: 돈이 관련된 화면에서 리스크 정보가 스크롤 없이 보이는가?

### P4. Progressive Disclosure — 처음엔 단순, 필요하면 확장

초보 유저에게 모든 걸 한 번에 보여주지 않는다.
- 기본: 핵심 정보 3개 이하
- 확장: "더 보기" / 탭 / 드릴다운으로 상세

적용 검증: 첫 화면에 정보 항목이 5개를 초과하면 설계를 의심한다.

### P5. One-Screen Decision — 핵심 의사결정은 1화면 내 완료

페이지 이동 없이, 스크롤 최소로, 한 화면에서 판단하고 액션까지.
- 나쁜 예: 차트 확인 → 별도 페이지에서 에이전트 확인 → 또 별도 페이지에서 트레이드
- 좋은 예: 차트 + 에이전트 요약 + 트레이드 버튼이 한 화면

적용 검증: "이 결정을 내리려면 다른 페이지에 갔다 와야 하는가?" Yes면 실패.

### P6. Trust Through Transparency — 숫자를 믿을 수 있게

트레이딩 앱에서 유저의 가장 큰 불안은 "이 데이터가 정확한가?", "이 액션이 내가 의도한 대로 실행되나?"이다.
- 가격 옆에 데이터 소스 + 최종 업데이트 시각 표시 (예: "Binance · 2초 전")
- 트레이드 실행 전 확인 화면에서 예상 결과를 명시 (예: "BTC LONG × $100 — 예상 청산가: $94,200")
- 에이전트 추천 옆에 신뢰도 근거 표시 (예: "72% confidence — 최근 10판 중 7판 적중")

적용 검증: 화면에 숫자가 있으면 "이게 언제 기준이고 어디서 온 건지" 유저가 알 수 있는가?

## 3. 유저 세그먼트 — "편함"의 기준이 다른 3명

| 세그먼트 | 특성 | "편함"의 의미 | 빈도 |
|----------|------|-------------|------|
| **Beginner** | 크립토 처음, 차트 못 읽음, 용어 모름 | "뭘 해야 하는지 알려줘" — 가이드 + 추천 중심 | 첫 1~10판 |
| **Active** | 기본 이해 있음, 주 3~5회 사용 | "빠르게 확인하고 실행" — 속도 + 원클릭 중심 | 10~100판 |
| **Pro** | 모든 지표 이해, 매일 사용, 자기 전략 있음 | "내 방식대로 커스텀" — 정보 밀도 + 단축키 중심 | 100판+ |

**설계 규칙: 기본 화면은 Active 기준으로 설계한다.** Beginner는 컨텍스트 가이드(인라인 힌트)로 보완하고, Pro는 밀도 모드/단축키로 확장한다.

각 페이지별 세그먼트 대응:

| 페이지 | Beginner 보완 | Active 기본 | Pro 확장 |
|--------|-------------|------------|---------|
| Terminal | 지표 옆 "이게 뭔가요?" 툴팁 | 차트 + Verdict + 액션 | 밀도 모드, 멀티 패널, 단축키 |
| Arena | 첫 3판 가이드 오버레이 | Quick Match 원클릭 시작 | 커스텀 스쿼드, 가중치 슬라이더 |
| Signals | "이 시그널은 이런 뜻" 카드 하단 설명 | 카드 스캔 + Track/Trade | 필터, 정렬, AI 랭킹 상세 |
| Passport | "여기서 성적표를 봅니다" 온보딩 포인터 | 헤더 4개 숫자 즉시 | 에이전트별 상세, 캘리브레이션 |

## 4. 유저 태스크 맵 — 유저가 실제로 하려는 일 5가지

모든 기능은 이 5가지 태스크 중 하나를 빠르게 완료시키기 위해 존재한다.

| # | 태스크 | 유저 질문 | 목표 시간 | 주요 페이지 |
|---|--------|----------|----------|------------|
| T1 | **시장 파악** | "지금 시장 어때?" | < 3초 | Terminal |
| T2 | **판단 + 액션** | "이거 사야해? 팔아야해?" | < 30초 | Terminal, Arena |
| T3 | **성과 확인** | "내가 얼마 벌었어?" | < 3초 | Passport |
| T4 | **AI 참고** | "AI는 뭐라고 해?" | < 5초 | Terminal, Arena, Agents |
| T5 | **커뮤니티 탐색** | "다른 사람들은 뭐 하고 있어?" | < 5초 | Signals |

**설계 원칙: 모든 페이지는 위 태스크 중 최소 1개를 담당하고, 해당 태스크의 목표 시간을 초과하면 안 된다.**

## 5. 페이지별 Usability 설계

---

### 5.1 Home (`/`)

**User Job**: 이 서비스가 뭔지 이해하고, 내가 할 일이 있는 곳으로 빠르게 이동한다.

**Arrival Intent**:
- 신규: "이게 뭐지?" → 제품 약속 이해 → Terminal이나 Arena로 이동
- 재방문: Home을 거치지 않고 직접 Terminal/Arena로 간다 (Home은 허브가 아님)

**Critical Path** (신규 유저):
```
랜딩 → Hero 읽기(5초) → CTA 클릭(1회) → Terminal 도착
총: 1클릭, 10초 이내
```

**First Screen에 반드시 있을 것**:
- 1줄 가치 제안 (읽기 3초 이내)
- Primary CTA → `/terminal` (지갑 없이 진입 가능)
- Secondary CTA → `/arena` 또는 지갑 연결

**First Screen에 없어야 할 것**:
- 긴 기능 설명 텍스트
- 가입 강제 (이메일/지갑 없이도 Terminal 진입 가능해야 함)
- 스크롤해야 보이는 CTA

**Mobile 차이**:
- Hero 텍스트 더 짧게 (2줄 이내)
- CTA 버튼 thumb zone (화면 하단 40%)에 위치

**Zero State** (첫 방문, 데이터 없음):
- Hero + CTA만 보임 — 추가 데이터 불필요
- "로그인 없이 시작" 강조 — 진입 장벽 제로

**Failure Recovery**:
- 지갑 연결 실패 시: Terminal CTA는 항상 작동 (지갑 무관)
- 로딩 지연 시: Hero 텍스트 + CTA는 JS 없이도 렌더

### 5.2 Terminal (`/terminal`)

**User Job**: 시장 맥락을 빠르게 파악하고, 액션할지 결정한다.

**담당 태스크**: T1(시장 파악), T2(판단+액션), T4(AI 참고)

**Arrival Intent**:
- "지금 BTC 어때?" → 차트 + 핵심 지표 즉시 확인
- "스캔 결과 보고 트레이드할지 결정" → 스캔 요약 + 액션 버튼
- "시그널 왔는데 어떻게 해?" → 시그널 컨텍스트 + Track/Trade

**Critical Path**:
```
Terminal 진입 → 차트 + 핵심 지표 즉시 로드(T1: 3초)
              → 스캔/에이전트 요약 확인(T4: +2초)
              → Track or Trade 버튼(T2: +1클릭)
총: 1~2클릭, 10초 이내에 판단 완료
```

**First Screen (Above the Fold)**:
```
┌─────────────────────────────────────────────┐
│ [BTC ▲ $96,420 +1.2%]  [ETH] [SOL] [pair▼] │  ← 가격 즉시
├─────────────────────────────────────────────┤
│                                             │
│            차트 (60% 높이)                   │  ← T1 즉시 해결
│                                             │
├─────────────────────────────────────────────┤
│ Agent Verdict: LONG 72%                     │  ← T4 즉시 해결
│ "OI 안정 + 고래 유입 + 추세 정배열"          │  ← P2 Explain Always
│                                             │
│ [Track] [Quick Trade] [Copy Trade] [Share]  │  ← T2 액션 (P1 Action First)
└─────────────────────────────────────────────┘
```

**On Demand (스크롤/탭/확장)**:
- War Room 상세 (에이전트별 개별 분석)
- Intel 패널 (뉴스, 이벤트, 고래 흐름)
- 커뮤니티 채팅
- 포지션 관리 패널

**Desktop vs Tablet vs Mobile**:
- Desktop: 3패널 (차트 + War Room + Intel) 동시
- Tablet: 차트 상단 + 하단 탭 (War Room / Intel)
- Mobile: 차트 전체 + 하단 고정 액션 바 + 탭 스와이프

**Mobile Critical Rule**:
- 차트와 액션 바는 항상 보임 (스크롤해도)
- War Room/Intel은 바텀 시트로 올라옴
- 엄지로 닿는 영역에 Track/Trade 버튼

**Zero State** (첫 Terminal 진입, 포지션/스캔 없음):
- 차트는 BTC/USDT 기본 페어로 즉시 렌더 (데이터 의존 없음)
- Agent Verdict 영역: "아직 스캔 데이터가 없습니다 — [첫 스캔 실행하기]" CTA
- 포지션 패널: "열린 포지션 없음 — 첫 트레이드는 여기서 시작" + Quick Trade 하이라이트
- War Room: 에이전트 카드 8개는 메타데이터로 표시, 성과 데이터 없으면 "분석 준비 중" 상태

**Failure Recovery**:
- 가격 로딩 실패: 마지막 캐시 가격 + "⏳ 업데이트 중" 표시
- 스캔 실패: "스캔 불가 — 차트와 수동 분석으로 판단하세요" + 액션 버튼은 유지
- 지갑 미연결 상태에서 Trade 클릭: 즉시 지갑 연결 모달 (트레이드 의도 보존)

### 5.3 Arena (`/arena`, `/arena-war`, `/arena-v2`)

**User Job**: 내 해석과 AI 해석을 비교하고, 그 결과로 배운다.

**담당 태스크**: T2(판단+액션), T4(AI 참고)

**Arrival Intent**:
- "연습 한 판 하자" → 빠르게 매치 시작
- "내 판단이 맞는지 AI랑 비교하고 싶다" → 가설 입력 → 비교
- "배틀 결과 보자" → 히스토리에서 리플레이

**현재 문제**: Arena가 3개 라우트(`/arena`, `/arena-v2`, `/arena-war`)로 분산. 유저가 어디로 가야 하는지 모름.

**Usability 설계 원칙**:
```
진입점은 1개: /arena
모드 선택은 Arena 안에서: [Quick Match] [Arena War] [Tournament]
```

**Critical Path (Quick Match)**:
```
/arena 진입 → 모드 선택(1클릭) → 스쿼드 선택(1클릭, 추천 프리셋 제공)
            → 가설 입력(방향 + 신뢰도, 15초) → AI 분석 대기(5초)
            → 결과 비교(즉시)
총: 3클릭 + 20초 입력, 30초 이내에 한 판 완료
```

**First Screen**:
```
┌─────────────────────────────────────────────┐
│ Arena                          [History 📋]  │
├─────────────────────────────────────────────┤
│                                             │
│  [⚡ Quick Match]    가장 빠른 1판           │  ← 기본 추천
│  [⚔️ Arena War]     AI와 정면 대결          │
│  [🏆 Tournament]    시즌 랭킹               │
│                                             │
├─────────────────────────────────────────────┤
│ 최근 전적: 7승 3패 (70%)  |  LP: 1,240     │
│ 추천 스쿼드: DERIV + STRUCTURE + MACRO      │  ← P4 Progressive Disclosure
│ "최근 5판에서 이 조합 80% 승률"              │  ← P2 Explain Always
└─────────────────────────────────────────────┘
```

**Phase 진행 중 UX 규칙**:
- 현재 Phase 번호 + 남은 Phase가 항상 상단에 표시
- 각 Phase에서 유저가 해야 할 행동이 1가지로 명확 (2가지 이상이면 설계 실패)
- Phase 전환 시 0.3초 이내 렌더 (로딩 스피너 금지, 스켈레톤 허용)

**Mobile Critical Rule**:
- Phase 진행 바: 상단 고정
- 입력 영역: 화면 하단 50% (키보드 위)
- 결과 화면: 핵심 3줄 → 상세는 "더 보기"

**Zero State** (첫 Arena 진입, 전적 없음):
- 전적 영역: "아직 전적이 없습니다" 대신 "첫 판을 시작해 보세요!" + Quick Match 버튼 강조
- 추천 스쿼드: 기본 프리셋 3개 제공 ("밸런스형", "공격형", "방어형") + "왜 이 조합?" 설명
- 스쿼드 선택 화면: 에이전트별 1줄 역할 설명 표시 (Beginner 대응)
- 모드 설명: 각 모드 카드에 "이런 사람에게 추천" 한 줄 (Quick Match: "처음이라면 여기부터")

**Failure Recovery**:
- 서버 sync 실패: 로컬 모드로 자동 전환 + "오프라인 모드" 배지
- 에이전트 분석 타임아웃: "분석 지연 — 결과 없이 진행하시겠습니까?" + Skip 버튼
- 중간 이탈(뒤로가기): 매치 상태 보존 → 재진입 시 이어서 (5분 TTL)

### 5.4 Signals (`/signals`)

**User Job**: 다른 사람들의 시그널을 발견하고, 관심 있으면 추적하거나 트레이드로 전환한다.

**담당 태스크**: T5(커뮤니티 탐색), T2(판단+액션)

**Arrival Intent**:
- "다른 사람들 뭐 하고 있어?" → 피드 스캔
- "좋은 시그널 있나?" → 트렌딩 확인
- "AI 에이전트 순위 보자" → AI 리더보드

**Critical Path**:
```
/signals 진입 → 피드 첫 3개 카드 즉시 로드(T5: 3초)
              → 관심 카드 탭(1클릭) → [Track] or [Trade](1클릭)
총: 2클릭, 10초 이내
```

**First Screen**:
```
┌─────────────────────────────────────────────┐
│ [Feed]  [Trending]  [AI Ranking]            │  ← 탭 3개만
├─────────────────────────────────────────────┤
│ 📊 @CryptoKim — BTC LONG 72%               │
│ "OI 안정 + 고래 유입, 단기 상승 기대"        │  ← P2 Explain
│ +2.3% since posted  |  12 tracks            │
│ [Track] [→ Trade]                    ❤️ 24  │  ← P1 Action First
├─────────────────────────────────────────────┤
│ 📊 @TraderJoe — ETH SHORT 65%              │
│ ...                                         │
└─────────────────────────────────────────────┘
```

**카드 정보 우선순위** (위에서 아래로):
1. 누가, 어떤 코인, 어떤 방향 (1줄)
2. 왜 (이유 1줄)
3. 현재 성과 (포스팅 이후 수익률)
4. 액션 버튼 (Track / Trade)
5. 소셜 지표 (좋아요, 댓글) — 가장 낮은 우선순위

**Mobile Critical Rule**:
- 카드 높이 최대 120px (스크롤 없이 2~3개 보임)
- Track 버튼은 카드 우측에 항상 노출 (스와이프 액션 아님)
- → Trade 누르면 Terminal로 이동 시 pair/direction/levels 자동 채움

**Zero State** (첫 Signals 진입, 추적 시그널 없음):
- 피드는 즉시 표시 — 퍼블릭 데이터이므로 Zero State가 발생하지 않음
- Tracked 필터 선택 시: "아직 추적 중인 시그널이 없습니다 — 피드에서 관심 시그널을 [Track] 해보세요"
- AI Ranking 탭: 에이전트 목록은 항상 표시 (서버 메타데이터), 성과 없으면 "데이터 수집 중"

**Failure Recovery**:
- 피드 로딩 실패: "피드를 불러올 수 없습니다 — 새로고침" + 캐시된 마지막 피드 표시
- Track 실패: 토스트 "추적 실패 — 다시 시도" + 버튼 재활성화
- Terminal 핸드오프 데이터 유실: 최소한 pair만 전달, 나머지는 Terminal에서 수동 입력

### 5.5 Passport (`/passport`)

**User Job**: 내 스탯, 성장, 포지션을 한 곳에서 즉시 확인한다.

**담당 태스크**: T3(성과 확인)

**Arrival Intent**:
- "내가 얼마 벌었어?" → 총 PnL 즉시
- "내 승률이 얼마지?" → 전적 요약 즉시
- "열린 포지션 뭐 있어?" → 오픈 포지션 목록
- "프로필 수정하고 싶다" → 아바타/닉네임 편집

**Critical Path**:
```
/passport 진입 → 헤더에 PnL + 승률 + 티어 즉시 표시(T3: 2초)
               → 탭으로 상세 드릴다운(1클릭)
총: 0클릭(핵심 지표), 1클릭(상세)
```

**First Screen (Above the Fold)**:
```
┌─────────────────────────────────────────────┐
│ 🧑 CryptoKim          Gold ⭐  LP: 1,240   │
├─────────────────────────────────────────────┤
│  총 PnL        승률        매치          포지션│
│  +$2,340      70%         47전          3 open│  ← T3 즉시 해결 (0클릭)
│  ▲ +12% MTD   33W 14L     7연승         ──── │
├─────────────────────────────────────────────┤
│ [Wallet]  [Positions]  [Profile]  [Arena]   │
└─────────────────────────────────────────────┘
```

**헤더 정보 우선순위**:
1. 총 PnL (금액) — 가장 큰 폰트
2. 승률 + 전적 — 두 번째
3. 티어 + LP — 세 번째
4. 오픈 포지션 수 — 네 번째

**On Demand (탭 진입 후)**:
- Wallet: 보유 자산 상세, 체인별 잔고
- Positions: 오픈/클로즈 트레이드 목록, 개별 PnL
- Profile: 아바타/닉네임 편집, 배지, 캘리브레이션
- Arena: 에이전트별 전적, 학습 파이프라인 상태

**Mobile Critical Rule**:
- 헤더 4개 숫자는 2×2 그리드로 항상 Above the Fold
- 탭 전환은 스와이프 지원
- 프로필 편집은 인라인 (별도 페이지 이동 금지)

**Zero State** (첫 Passport 진입, 활동 기록 없음):
- 헤더 4개 숫자: 모두 "—" 대신 "0" 표시 (PnL: $0, 승률: —, 매치: 0전, 포지션: 0 open)
- "아직 활동 기록이 없습니다" + 행동 유도 CTA 2개:
  - "첫 트레이드 해보기 → Terminal"
  - "AI와 첫 대결 → Arena"
- Wallet 탭: 지갑 미연결 시 "지갑을 연결하면 보유 자산을 확인할 수 있습니다 — [연결하기]"
- Positions 탭: "아직 열린 포지션이 없습니다" + Terminal로 가는 CTA
- Profile 탭: 기본 아바타 + "닉네임을 설정해 보세요" 인라인 편집 하이라이트

**Failure Recovery**:
- 서버 프로필 로딩 실패: 로컬 캐시 먼저 표시 + "동기화 중..." 배지
- 홀딩스 없음: "아직 자산이 없습니다" + 지갑 연결 CTA (데모 데이터 명시)
- PnL 계산 지연: 마지막 알려진 값 + "⏳" 아이콘

### 5.6 Agents (`/agents`)

**User Job**: 에이전트가 누구이고 얼마나 잘하는지 빠르게 파악한다.

**담당 태스크**: T4(AI 참고)

**Arrival Intent**:
- "이번에 어떤 에이전트 쓸까?" → 에이전트별 최근 성과 비교
- "DERIV가 요즘 어때?" → 특정 에이전트 상세

**Critical Path**:
```
/agents 진입 → 8개 에이전트 카드 그리드 즉시 로드(3초)
             → 카드 탭(1클릭) → 상세 패널 오픈
총: 1클릭, 5초 이내
```

**First Screen**:
```
┌─────────────────────────────────────────────┐
│ Agents                    [Sort: 승률순 ▼]  │
├──────────────────┬──────────────────────────┤
│ 💰 DERIV         │ 📊 STRUCTURE             │
│ ★★★ Squeeze     │ ★★ Trend Rider          │
│ 승률 72% | 49전  │ 승률 65% | 52전          │
│ 최근: ▲▲▲▼▲    │ 최근: ▲▼▲▲▼            │
├──────────────────┼──────────────────────────┤
│ ⚡ ICT           │ 🐋 FLOW                  │
│ ...              │ ...                      │
└──────────────────┴──────────────────────────┘
```

**카드 정보 우선순위**:
1. 이름 + 현재 Spec (1줄)
2. 승률 + 전적 수 (핵심 판단 지표)
3. 최근 5판 결과 (▲▼ 시각화)
4. 티어/레벨 — 낮은 우선순위

**On Demand (카드 탭 후)**:
- 패턴 메모리 목록
- 시장 레짐별 성과
- 매치업 데이터
- 학습 레벨 상세

**Zero State** (첫 Agents 진입, 매치 기록 없음):
- 8개 에이전트 카드 그리드는 항상 표시 (서버 메타데이터: 이름, 역할, 아이콘)
- 승률/전적: "아직 데이터 없음" 대신 "첫 매치 후 성과가 표시됩니다"
- 최근 5판 시각화: 빈 상태 → 회색 점 5개 (데이터 채워지면 ▲▼로 전환)
- 카드 탭 시 상세: 메타데이터(역할 설명, 분석 영역)는 표시, 성과 섹션만 "매치 후 업데이트"

**Mobile Critical Rule**:
- 2열 그리드 (카드 4개가 화면에 보임)
- 상세는 바텀 시트 (전체 페이지 전환 아님)

### 5.7 Settings (`/settings`)

**User Job**: 내 환경을 빠르게 바꾸고 저장되었는지 확인한다.

**Critical Path**: 진입 → 설정 변경 → 자동 저장 확인. 저장 버튼 불필요 (자동 저장 + "✓ 저장됨" 피드백).

**핵심 규칙**:
- 설정 변경 즉시 반영 (페이지 새로고침 불필요)
- 클라우드 동기화 상태 항상 표시 ("클라우드 저장됨" / "로컬만")
- 위험한 액션(데이터 초기화)은 별도 확인 모달

## 6. 크로스 페이지 플로우 — 페이지 간 이동 최적화

### 6.1 시그널 → 트레이드 플로우 (가장 빈번)

```
Signals 카드에서 [→ Trade] 클릭
    ↓ pair + direction + entry + tp + sl + confidence + reason 자동 전달
Terminal 도착 → Copy Trade 모달 자동 오픈 (프리필 완료)
    ↓ 유저는 확인만 하고 [실행]
```

**규칙**: 핸드오프 시 최소 pair는 반드시 전달. 나머지 유실 시 Terminal에서 수동 입력 가능.

### 6.2 Arena → 학습 플로우

```
Arena 결과 화면에서 "이 에이전트 더 보기" 링크
    ↓
Agents 상세로 이동 (해당 에이전트 자동 선택)
    ↓
"이 에이전트로 다시 매치" → Arena로 복귀 (스쿼드 프리셋)
```

### 6.3 공통 네비게이션 규칙

```
모든 페이지에서 접근 가능:
- Header: Home | Terminal | Arena | Signals | Passport
- Mobile Bottom Bar: Terminal | Arena | Signals | Passport
  (Home은 로고 탭)

현재 페이지 하이라이트 항상 표시.
Agents, Settings는 2차 네비게이션 (Passport 안 또는 메뉴).
```

### 6.4 딥링크 보존

모든 주요 상태는 URL에 반영:
- `/terminal?pair=BTC&tf=1h` — 페어 + 타임프레임
- `/signals?view=ai` — 탭 상태
- `/passport?tab=positions` — 탭 상태
- `/arena?mode=war` — 모드 선택

**이유**: 공유 가능 + 뒤로가기 시 상태 유지 + 북마크 가능.

## 7. 반응형 전략

### 7.1 브레이크포인트

| 이름 | 범위 | 레이아웃 전략 |
|------|------|-------------|

| Tablet | 768~1023px | 2컬럼 (주요 + 보조) |

### 7.2 모바일 필수 규칙 (모든 페이지)

1. **Primary CTA는 thumb zone** (화면 하단 40%)에 위치
2. **고정 바텀 네비게이션**: Terminal | Arena | Signals | Passport
3. **Above the Fold에 핵심 정보**: 스크롤 전에 태스크 답변 가능
4. **바텀 시트 > 새 페이지**: 상세 정보는 바텀 시트로 (페이지 전환 최소화)
5. **입력 시 키보드 위 영역 확보**: 가설 입력, 트레이드 입력 시 입력 필드가 키보드에 가려지지 않음

### 7.3 Desktop 필수 규칙

1. **다중 패널 동시 표시**: Terminal은 차트 + War Room + Intel 동시
2. **키보드 단축키**: Arena 뷰 전환 (1-4), Terminal 페어 전환 (←→)
3. **Hover 상태 활용**: 카드 hover 시 프리뷰 (모바일에선 없음)

## 8. 에러 & 복구 — 전역 규칙

### 8.1 에러 표시 원칙

| 심각도 | 표시 방식 | 예시 |
|--------|----------|------|
| **경고** | 인라인 배너 (노랑) | "가격 업데이트 지연 중" |
| **실패** | 토스트 + 재시도 버튼 | "트레이드 실패 — 다시 시도" |
| **차단** | 전체 화면 에러 + 안내 | "서버 연결 불가 — 오프라인 모드로 전환됨" |

### 8.2 오프라인/지연 대응

- 모든 읽기 화면: 마지막 캐시 데이터 표시 + "⏳ 업데이트 중"
- 쓰기 액션: "오프라인 — 연결 후 자동 제출됩니다" (optimistic staging)
- 5초 이상 로딩: 스켈레톤 → 10초 이상: "느린 연결" 메시지 + 취소 옵션

### 8.3 인증 에러

- 미로그인 상태에서 보호 액션 시도: 즉시 지갑/로그인 모달 (원래 의도 보존)
- 세션 만료: 현재 화면 유지 + 상단 배너 "세션 만료 — 다시 로그인" + 로컬 작업 보존

## 9. 데이터 Freshness 계약 — 숫자를 언제까지 믿을 수 있는가

트레이딩 앱에서 "오래된 가격"을 보고 판단하는 것은 직접적 손실로 이어진다. 모든 데이터에 freshness 계약을 명시한다.

### 9.1 Freshness 등급

| 등급 | 허용 지연 | 표시 규칙 | 예시 |
|------|----------|----------|------|
| **Real-time** | ≤ 2초 | 소스 + "n초 전" 라이브 카운터 | 현재가, 오더북, 펀딩레이트 |
| **Near-time** | ≤ 30초 | 소스 + 마지막 업데이트 시각 | OI, 고래 흐름, 스캔 결과 |
| **Periodic** | ≤ 5분 | 시각만 표시 | 에이전트 Verdict, 뉴스 요약 |
| **Snapshot** | 명시적 시점 | "YYYY-MM-DD HH:mm 기준" | PnL 정산, 리더보드 순위 |

### 9.2 Staleness 표시 규칙

```
정상 범위 내     → 소스 라벨만 (예: "Binance")
허용 지연 초과   → 노란 경고 "⚠ 15초 전 데이터" + 소스
2× 허용 지연 초과 → 빨간 경고 "🔴 데이터 오래됨 — 새로고침" + 액션 버튼 비활성화 검토
연결 끊김       → 마지막 캐시값 표시 + "⏳ 연결 복구 중" 배너
```

### 9.3 페이지별 적용

| 페이지 | Real-time 데이터 | Near-time 데이터 | 주의 사항 |
|--------|----------------|-----------------|----------|
| Terminal | 현재가, 차트 캔들 | OI, 고래 흐름, 스캔 | Trade 실행 직전 가격은 반드시 Real-time 확인 |
| Arena | 진행 중 매치 가격 | 에이전트 분석 | Phase 전환 시 가격 갱신 강제 |
| Signals | 시그널 포스팅 시점 가격 | 현재 성과 (since posted) | "포스팅 시점"과 "현재" 가격 구분 명시 |
| Passport | — | 포지션 PnL | "정산 기준 시각" 항상 표시 |

## 10. 알림 & 인터럽트 UX — 정보가 유저를 찾아가는 방식

### 10.1 알림 계층

| 긴급도 | 전달 방식 | 예시 | 중단 허용 |
|--------|----------|------|----------|
| **Critical** | 풀스크린 모달 + 사운드 | 청산 임박, 대규모 손실 경고 | 즉시 중단, 확인 필수 |
| **Important** | 인앱 배너 (상단 고정, 자동 사라지지 않음) | 스캔 완료, 시그널 트리거, 포지션 TP/SL 도달 | 현재 작업 중단 안 함, 눈에 보임 |
| **Informational** | 토스트 (3초 자동 사라짐) | Track 성공, 설정 저장됨, 새 커뮤니티 포스트 | 중단 없음 |
| **Passive** | 배지/카운터 (네비게이션 아이콘) | 읽지 않은 시그널 수, 새 댓글 | 중단 없음, 발견형 |

### 10.2 인터럽트 규칙

1. **트레이드 실행 중에는 Critical 외 모든 알림 억제** — 주문 입력/확인 플로우 중에 토스트나 배너로 주의를 흐리면 실수 유발
2. **Arena Phase 진행 중에는 배너까지만 허용** — 모달은 Phase 완료 후 큐잉
3. **같은 유형 알림 쌓이지 않게** — 동일 소스 알림은 카운터로 합산 (예: "시그널 3개 업데이트" 하나)
4. **모바일: 진동은 Critical만** — 나머지는 시각적 알림만

### 10.3 스캐너 시그널 인터럽트 (Terminal 특화)

```
스캐너가 새 시그널 감지 시:
1. Terminal에 있을 때: Agent Verdict 영역에 인라인 업데이트 (깜빡임 1회 + 하이라이트 3초)
2. 다른 페이지에 있을 때: 네비게이션의 Terminal 아이콘에 빨간 점 + "새 시그널" 배지
3. 앱 밖에 있을 때: [향후] 푸시 알림 (유저 opt-in 필수)
```

## 11. 첫 사용자 경험 — Progressive Onboarding

### 11.1 원칙: 가르치지 말고 유도한다

- **강제 튜토리얼 금지** — 30초 이상 읽기만 해야 하는 온보딩은 이탈만 유발
- **행동 중에 가르침** — 유저가 실제로 뭔가 하려는 순간에 컨텍스트 힌트 제공
- **스킵 항상 가능** — 모든 가이드/힌트는 "✕" 또는 "건너뛰기"로 즉시 닫기 가능
- **한 번만 보여주기** — 닫은 힌트는 다시 나타나지 않음 (localStorage 기반 seen 플래그)

### 11.2 첫 방문 플로우 (지갑 미연결 상태)

```
Home 도착
  → Hero 읽기 (3초)
  → "지갑 없이 시작하기" CTA → Terminal 도착
  → Terminal에서 첫 번째 컨텍스트 힌트:
    💡 "이 차트는 BTC 실시간 가격입니다. 아래 에이전트가 분석해 드립니다."
  → 유저가 차트/Verdict 탐색 (자유)
  → Agent Verdict 영역에 두 번째 힌트:
    💡 "이 점수는 AI 에이전트 8명의 종합 판단입니다. [자세히 알아보기]"
  → 탐색 후 자연스럽게 다른 페이지 이동 (강제 유도 없음)
```

### 11.3 마일스톤 기반 힌트 트리거

| 마일스톤 | 트리거 | 힌트 내용 | 위치 |
|----------|--------|----------|------|
| 첫 Terminal 방문 | Terminal 로드 | "차트 + 에이전트 분석 = 시장 한눈에 보기" | 차트 상단 |
| 첫 스캔 결과 확인 | Verdict 영역 노출 3초 | "Track으로 추적, Trade로 즉시 실행" | 액션 바 옆 |
| 첫 Arena 진입 | Arena 로드 | "Quick Match로 AI와 한 판 — 3분이면 끝!" | 모드 선택 영역 |
| 첫 매치 완료 | 결과 화면 | "결과가 Passport에 기록됩니다 — 전적 확인해 보세요" | 결과 하단 |
| 첫 시그널 Track | Track 버튼 클릭 | "24시간 동안 추적됩니다. Signals에서 확인 가능" | 토스트 |
| 지갑 연결 시도 | Trade 또는 Wallet CTA 클릭 | "지갑을 연결하면 실제 포지션 추적이 가능합니다" | 모달 내 |

### 11.4 재방문 유저 처리

- 2회차 이후 방문: 모든 힌트 숨김, Home 대신 Terminal 또는 마지막 활동 페이지로 직접 진입
- 30일 이상 미접속 후 복귀: "돌아오셨군요! 마지막 접속 이후 변경사항:" 요약 카드 (1회성)

## 12. 상태 복원 — App Resume

### 12.1 원칙: 유저가 떠났던 곳에서 다시 시작

앱을 닫았다 열면 처음부터 다시 시작해야 하는 경험은 마찰이다.

### 12.2 복원 계약

| 상태 | 저장 위치 | 복원 조건 | TTL |
|------|----------|----------|-----|
| **마지막 방문 페이지 + 탭** | localStorage | 항상 | 무제한 |
| **Terminal 페어 + 타임프레임** | URL + localStorage | 항상 | 무제한 |
| **Arena 진행 중 매치** | localStorage + 서버 | 앱 재진입 시 | 5분 |
| **Signals 필터/정렬 상태** | URL params | 항상 (URL 기반) | 무제한 |
| **Passport 활성 탭** | URL params | 항상 | 무제한 |
| **Trade 입력 중 값** | sessionStorage | 같은 세션 내 탭 전환 | 탭 닫으면 초기화 |
| **스크롤 위치** | — | 복원하지 않음 | — |

### 12.3 복원 UX 규칙

1. **복원은 무음으로** — "이전 상태를 복원했습니다" 같은 메시지 불필요 (자연스럽게)
2. **예외: Arena 매치 중단 복원** — "이전 매치가 남아 있습니다 — [이어하기] [새 매치]" 확인 필요
3. **서버 데이터와 충돌 시** — 서버가 우선. 로컬 캐시는 렌더 후 서버 응답으로 교체 (FOUC 최소화)
4. **크로스 디바이스** — 서버 저장 상태(포지션, 전적, 프로필)는 디바이스 무관 복원. UI 상태(페어, 탭)는 디바이스별.

## 13. 트레이딩 도메인 특화 패턴

### 13.1 슬리피지 & 실행 경고

트레이드 실행 직전에 유저가 예상과 다른 결과를 받을 수 있는 상황을 사전 경고:

| 상황 | 표시 | 차단 여부 |
|------|------|----------|
| 현재가가 입력 시점 대비 0.5%+ 변동 | "⚠ 가격이 변동했습니다: $96,420 → $96,900" + [갱신 후 실행] | 차단 아님, 경고만 |
| 주문 크기가 유동성 대비 과대 | "⚠ 현재 유동성 대비 주문이 큽니다 — 슬리피지 예상" | 경고 + 확인 필요 |
| 펀딩레이트가 -0.1% 이하 (LONG) 또는 +0.1% 이상 (SHORT) | "💰 불리한 펀딩레이트: -0.15% — 보유 비용 증가" | 경고만 |

### 13.2 대규모 트레이드 확인

포지션 크기가 일정 기준을 초과하면 추가 확인 단계:

```
기본 트레이드: [실행] 버튼 1번 클릭
대규모 트레이드 (계좌 대비 20% 이상):
  → 확인 모달: "이 주문은 계좌의 25%입니다"
  → 예상 청산가, 최대 손실 금액 명시
  → [확인 후 실행] 버튼 (텍스트 변경으로 심리적 마찰 추가)
```

### 13.3 가격 신선도 인디케이터

Terminal 차트 옆 가격 표시에 실시간 freshness:

```
● 초록 점 + "Binance · 1초 전"     → 정상
● 노란 점 + "Binance · 8초 전"     → 경고 (Real-time 허용 초과)
● 빨간 점 + "마지막 가격 · 30초 전"  → 가격 신뢰 불가 (Trade 버튼 비활성화 검토)
```

### 13.4 포지션 위험 시각화

열린 포지션의 위험도를 색상 + 거리로 즉시 인지:

```
┌─────────────────────────────────┐
│ BTC LONG  +$240 (+2.4%)        │  ← 초록 (이익)
│ ████████████████░░░░  TP까지 60%│
│ Entry: 94,200  SL: 93,000  TP: 96,200 │
├─────────────────────────────────┤
│ ETH SHORT  -$85 (-1.7%)        │  ← 빨강 (손실)
│ ░░░░░░░░████████████  SL까지 30%│
│ Entry: 3,420  SL: 3,500  TP: 3,200   │
└─────────────────────────────────┘
```

- TP/SL까지의 거리를 프로그레스 바로 시각화
- 손절 거리 30% 이내: 노란색 경고
- 손절 거리 10% 이내: 빨간색 + "⚠ 청산 임박"

### 13.5 Copy Trade 안전장치

시그널에서 Copy Trade로 전환할 때:

1. 원본 시그널의 entry 가격과 현재 가격 차이 표시: "시그널 당시 $94,200 → 현재 $96,420 (+2.4%)"
2. "이미 상당히 진입한 시그널입니다" 경고 (entry 대비 2%+ 이동 시)
3. 포지션 크기는 유저의 설정/한도에 맞게 자동 조정 (시그널 작성자의 크기 그대로 복사하지 않음)
4. TP/SL은 시그널 값을 프리필하되, 유저가 반드시 확인 후 실행

## 14. 접근성 & 인지 부하 관리

### 14.1 색상 접근성

- 이익/손실: 초록/빨강 외에 ▲/▼ 아이콘 + 부호(+/-) 병행 → 색각 이상 대응
- 차트 라인: 색상만으로 구분 금지 → 실선/점선/대시 등 패턴 병행
- 대비율: 모든 텍스트-배경 조합 WCAG AA (4.5:1) 이상

### 14.2 인지 부하 한도

| 영역 | 한도 | 초과 시 |
|------|------|--------|
| First Screen 정보 항목 | ≤ 5개 | Progressive Disclosure 적용 |
| 모달 내 선택지 | ≤ 3개 | "더 보기"로 확장 |
| 동시 애니메이션 | ≤ 2개 | 나머지 큐잉 |
| 토스트 동시 표시 | ≤ 1개 | 스택 (최신만 보이고 이전은 카운터) |
| 네비게이션 최상위 항목 | ≤ 5개 | 2차 메뉴로 이동 |

### 14.3 로딩 인지 패턴

```
0~200ms: 즉시 렌더 (로딩 표시 없음)
200ms~1초: 스켈레톤 UI
1초~5초: 스켈레톤 + 프로그레스 표시
5초~10초: "연결이 느립니다" 메시지 + 취소 옵션
10초+: 에러 상태로 전환 + 재시도 버튼
```

## 16. UserJourney v1에서 선별 차용할 것

Engagement 레이어는 usability 위에 얹는다. v1에서 가져올 것:

| 항목 | 적용 위치 | 적용 방식 |
|------|----------|----------|
| LP + 티어 진행 | Passport 헤더 | 현재 LP + 다음 티어까지 남은 양 표시 |
| 연패 방지 | Arena 결과 화면 | 3연패 시 조합 변경 제안 (강제 아님) |
| 해금 타임라인 | Agents 카드 | "Spec C까지 2전!" (이미 있는 데이터에 레이블만) |
| 감정 곡선 대응 | 첫 DISSENT 시 | 1회성 툴팁 "에이전트가 항상 맞지 않습니다" |

**가져오지 않을 것**:
- 강제 온보딩 30초 애니메이션 → 스킵 가능하거나 제거
- 60초 드래프트 타이머 → Quick Match에서는 시간 제한 없음
- Spec A/B/C 복잡한 해금 트리 → Progressive Disclosure로 단순화
